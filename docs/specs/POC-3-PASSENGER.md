## Цель и выбранный подход

Доказать: **игрок стоит, ходит и прыгает на движущемся ровере**. Подход —
честная кинематическая капсула, а не «магнитное» приклеивание:

1. Игрок становится `CharacterBody3D` с капсулой (radius 0.35, height 1.8).
2. Езда на носителе — штатный механизм движущихся платформ Godot:
   `move_and_slide` сам учитывает скорость пола-`RigidBody3D` (включая
   вращательную составляющую в точке контакта).
3. Наследование скорости при прыжке/сходе — встроенное:
   `platform_on_leave = PLATFORM_ON_LEAVE_ADD_VELOCITY`.
4. Сцепление конечное: при резком манёвре носителя капсула может поехать —
   это желаемое поведение, а не баг.

Архитектурная страховка: понятие «фрейм опоры» оформить отдельным компонентом
`scripts/support_frame.gd`, чтобы позже можно было заменить реализацию на
симуляцию в локальном фрейме грида (как в Space Engineers) без переписывания
контроллера. В v0 компонент только сообщает носителя и скорость точки опоры.

## Слои коллизии (обязательная схема)

| Layer | Кто |
|---|---|
| 1 | мир: террейн, пол тестов, статичные блоки `PlacedBlocks` |
| 2 | техника: `Cart`, `LaunchVehicle`, `Assembly`, обломки, оторванные колёса |
| 3 | капсула игрока и тест-пробы |

Маски:

- игрок: mask 1|2 (сталкивается с миром и техникой);
- техника: layer 2, mask 1|2 (**не** 3 — игрок не может заблокировать или
  перевернуть ровер своим телом; односторонняя коллизия);
- suspension-raycast в `cart.gd`: `collision_mask = 1|2` (не ловить капсулу);
- raycast'ы бура и block placer: исключить `get_rid()` капсулы игрока.

`VoxelTerrain` оставляет дефолтный layer 1.

## Задача 1 — перевод игрока на CharacterBody3D

`scenes/player.tscn`: корень → `CharacterBody3D` (layer 3, mask 1|2) +
`CollisionShape3D` с `CapsuleShape3D` (radius 0.35, height 1.8). Остальные
дети (Camera, Drill, BlockPlacer, Body, VoxelViewer) не меняются.

`scripts/player_controller.gd` переписать:

- `extends CharacterBody3D`;
- сохранить публичный API: `set_spawn_locked`, `set_spawn_ready`,
  `is_spawn_ready`, `is_grounded` (→ `is_on_floor()`), `enter_vehicle`,
  `exit_vehicle` (reparent в кокпит остаётся как есть; при входе отключать
  коллизию капсулы, при выходе включать);
- движение: ходьба задаёт горизонтальную `velocity` (speed 8.0, sprint ×1.6),
  гравитация игрока `22.0` вручную (мировую 1.62 НЕ использовать — геймплейное
  решение), прыжок `jump_velocity = 8.5` (высота ≈ 1.64 м — хватает на палубу
  1.5 м; старых 7.5 не хватало);
- `platform_on_leave = PLATFORM_ON_LEAVE_ADD_VELOCITY`,
  `floor_snap_length = 0.4`, `floor_max_angle ≈ 60°`;
- voxel-fallback уходит: пол теперь честная коллизия террейна
  (`generate_collisions = true` уже включён). Спавн-лок до готовности террейна
  остаётся прежним;
- компонент `SupportFrame` (child Node, скрипт `support_frame.gd`):
  - `carrier() -> RigidBody3D` — `get_platform_velocity()` не даёт тело, поэтому
    носителя определять из `get_last_slide_collision()` / floor collider;
  - `point_velocity(world_point) -> Vector3` —
    `v + ω × (point - com_world)` носителя;
  - контроллер сам его почти не использует в v0 (платформы делает движок), но
    компонент обязан существовать и публиковать носителя — это интерфейс для
    будущего локального фрейма и для тестов.

## Задача 3 — headless-автотест

`scenes/test_passenger.tscn` + `scripts/test_passenger.gd`:

- пол `StaticBody3D` 200×1×200 (layer 1), верх y = 0;
- `Cart`: `(0, 3, 0)`, `freeze = false`, `accept_player_input = false`;
- проба над палубой: `(0, 4.3, 0)`. При начальном origin ровера y=3 верх
  рамы y≈3.3, а низ капсулы на 0.9 м ниже её origin; 4.3 исключает стартовое
  пересечение и позволяет честно осесть на палубу.

Этапы (`POC3: PASS` / `POC3: FAIL <причина>`, `quit(0|1)`):

1. **Осадка** (0–3 с): проба на полу палубы: `is_on_floor()`, носитель из
   `SupportFrame` == ровер.
2. **3a — разгон** (3–7 с): `set_drive_command(1.0, 0.0)`. Ровер уехал ≥ 8 м;
   дрейф пробы по палубе (XZ в локальных координатах ровера относительно
   стартовой точки) < 0.5 м; проба всё ещё на ровере.
3. **3b — поворот** (7–9 с): тяга 0.5, руль 1.0. Дрейф < 0.8 м, проба на
   ровере. Небольшое скольжение допустимо — конечное сцепление.
4. **3c — прыжок на ходу** (9 с): руль 0, тяга 1.0, `jump_requested` один тик.
   Через 1.2 с: проба снова на ровере, дрейф от точки прыжка < 1.0 м.
   Смысл порога: без наследования скорости (~5 м/с, полёт ~0.77 с) дрейф был
   бы ~4 м.
5. **3d — сход** (после приземления): тормоз до остановки, затем
   `walk_input` вбок 1.5 с — проба сходит с палубы на пол. Проверить: носитель
   пуст, проба на полу (origin y в 0.85–1.25), нет NaN.
6. Финал: печать `POC3: PASS`.

Ожидания по таймингу — через `await get_tree().create_timer(...)` +
`await get_tree().physics_frame` как в `test_cart_rebuild.gd`.
