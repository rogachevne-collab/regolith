# Спека PoC-2 — пересборка физического тела

Самодостаточное задание для агента-исполнителя. Контекст беседы не нужен.
Родительский документ — `docs/PHYSICAL-LANGUAGE.md` (разделы Assembly, Element,
Body, «Прочность, разрушение и ремонт»), читать не обязательно.

## Цель

Доказать третий физический риск языка: **состав конструкции меняется в рантайме,
а физическое тело пересобирается без телепорта и скачка импульса**. Конструкция —
сетка кубических элементов внутри одного `RigidBody3D`. Проверяются три эффекта:

- **2a** — снятие/добавление элемента: масса, центр масс и compound collider
  пересчитываются, тело в покое не дёргается;
- **2b** — снятие элемента на ходу: поза и скорость непрерывны;
- **2c** — разрыв связности: отсоединившаяся компонента становится отдельным
  телом со своей унаследованной скоростью.

Прочность, усталость, нагрузки, ремонт — НЕ входят (это следующие PoC). Разрыв
происходит только по явной команде.

## Обязательная граница приёмки

Отдельная кубическая `Assembly` ниже — только лабораторный **PoC-2a rebuild
primitive**. Она сама по себе не закрывает PoC-2.

PoC-2 считается пройденным только на живом ровере из `cart.gd`:

- составом рамы владеет отдельный `StructureModel`;
- `attach`/`detach` поступают в модель как упорядоченные структурные команды;
- модель публикует структурное событие, а физический код ровера проецирует его в
  массу, центр масс, compound collider и новые тела;
- во время пересборки продолжают работать raycast-подвеска, привод и руление;
- тест `test_cart_rebuild.tscn` ломает и приваривает блок на движущемся ровере,
  затем разрывает мост и проверяет новое тело, непрерывность движения и сохранение
  суммарной массы.
- колесо является функциональным структурным элементом: тест
  `test_wheel_detach.tscn` отрывает его на скорости, проверяет переход на три
  активные подвески, отдельное физическое тело колеса и наследование скорости.

Прямая правка физических нод из обработчика пользовательской команды не
засчитывается: только `command → StructureModel → event → physics projection`.

## Задача 1 — scripts/assembly.gd + scenes/assembly.tscn

`assembly.tscn`: корень `RigidBody3D` со скриптом `assembly.gd`, без статических
детей — вся структура строится кодом.

### Модель данных

- `BLOCK_SIZE := 1.0` (м), элемент — куб.
- Словарь `_elements: Dictionary` (ключ `Vector3i`, значение — Dictionary с
  `mass: float`, `visual: MeshInstance3D`, `collision: CollisionShape3D`).
- Масса элемента по умолчанию 50.0 кг.
- Связность — 6-соседство (грань к грани).
- Корень связности: ячейка `Vector3i.ZERO`; если её нет — любая оставшаяся.

### Публичный API (структурные команды)

```gdscript
func build_from(cells: Array[Vector3i]) -> void   # начальная сборка
func attach_element(cell: Vector3i, element_mass: float = 50.0) -> bool
func detach_element(cell: Vector3i) -> bool
func element_count() -> int
func total_mass() -> float
```

- `attach_element` отклоняет ячейку, если она занята или не имеет соседа
  (кроме самой первой ячейки при пустой сборке).
- `detach_element` удаляет элемент, спавнит его как **отдельную сборку из одного
  элемента** (см. наследование скоростей ниже), затем проверяет связность.
- Структурные мутации, вызванные во время физического колбэка, откладывать через
  `call_deferred`.

### Пересборка (после каждой мутации)

1. Пересчитать `mass` = сумма масс элементов.
2. `center_of_mass_mode = CENTER_OF_MASS_MODE_CUSTOM`,
   `center_of_mass` = взвешенный центр ячеек (локально). Инерцию оставить
   автоматической (`inertia = Vector3.ZERO`).
3. Каждый элемент несёт свой `CollisionShape3D` (`BoxShape3D` 1×1×1 в позиции
   ячейки) — удалить/добавить только затронутые, не пересоздавать всё.
4. Визуал: `BoxMesh` 0.94×0.94×0.94 на ячейку, серый металлик.
5. **Компенсация скорости при смещении центра масс.** Скорость тела задана в
   центре масс. Если мировой центр масс сместился с `c_old` на `c_new`:

   ```text
   linear_velocity += angular_velocity.cross(c_new - c_old)
   ```

   `global_transform` не трогать вообще. Это и есть «без телепорта».

### Разрыв связности

После `detach_element` — BFS от корня. Каждая несвязная группа ячеек:

1. Удаляется из этой сборки (с пересборкой по правилам выше).
2. Спавнится как новый экземпляр `assembly.tscn` в том же родителе:
   - `global_transform` нового тела = текущий transform старого (ячейки группы
     сохраняют мировые позиции — передать их через `build_from` с теми же
     локальными координатами; допустимо нормализовать индексацию, если мировые
     позы элементов не изменились);
   - `angular_velocity` = у старого тела;
   - `linear_velocity` = `v_old + ω_old × (c_group − c_old)`, где `c_group` —
     мировой центр масс группы, `c_old` — мировой центр масс старого тела до
     разрыва.

## Задача 2 — headless-автотест

`scenes/test_assembly.tscn` + `scripts/test_assembly.gd`: плоский пол
(`StaticBody3D`, `BoxShape3D` 60×1×60, верх на y = 0), одна сборка, этапы идут
последовательно в одной сцене. Скрипт печатает `POC2: PASS` /
`POC2: FAIL <причина>` и завершает `get_tree().quit(0|1)`.

Начальная сборка — балка 5×1×1: ячейки `(0..4, 0, 0)`, на высоте y = 2.

Этапы:

1. **Осадка** (0–4 с): сборка падает и успокаивается.
   Проверить: `linear_velocity.length() < 0.05`, масса 250.
2. **2a — attach/detach в покое** (4 с): `attach_element(Vector3i(2, 1, 0))`,
   через 1 с `detach_element(Vector3i(2, 1, 0))`.
   Проверить: масса вернулась к 250; за кадр пересборки origin сместился
   < 0.05 м; в мире появилось отдельное тело-обломок (одноэлементная сборка).
3. **2b — detach на ходу** (7 с): придать телу импульс
   `apply_central_impulse(Vector3(0, 0, -mass * 2.0))`, через 0.5 с
   `detach_element(Vector3i(4, 0, 0))` (крайний, связность не рвётся).
   Проверить: в кадре пересборки |Δposition| за кадр ≤ `скорость * dt * 2`;
   изменение `linear_velocity.length()` в момент пересборки < 15 %.
4. **2c — раскол** (10 с): `detach_element(Vector3i(2, 0, 0))` — средний
   элемент балки, остаются группы `(0,1)` и `(3)` + обломки.
   Проверить: исходное тело имеет 2 элемента (масса 100), в мире появилось
   новое тело с 1 элементом (ячейка 3) со скоростью, отличающейся от исходного
   тела < 0.5 м/с в момент спавна; суммарная масса всех сборок в сцене = 250.
5. Финал (14 с): ни у одного тела нет NaN в позиции. Печать вердикта.

Для подсчёта тел использовать группу: сборки добавляют себя в
`add_to_group("assemblies")`.

## Критерии приёмки

1. `./run.sh --headless res://scenes/test_cart_rebuild.tscn` →
   `POC2-ROVER: PASS`, код 0 — главный критерий.
2. `./run.sh --headless res://scenes/test_assembly.tscn` → `POC2: PASS`, код 0 —
   лабораторный primitive.
3. `./run.sh --headless res://scenes/test_wheel_detach.tscn` →
   `POC2-WHEEL: PASS`, код 0.
4. Регресс: `./run.sh --headless res://scenes/test_cart_flat.tscn`,
   `.../test_cart_drive.tscn`, `.../test_cart_steering.tscn` остаются PASS.
5. В `./run.sh` `B`/`N` меняют раму движущегося ровера, `V` разрывает мост,
   `M` отрывает переднее левое колесо; оставшиеся подвески и привод работают.
6. Ни одной ошибки парсинга GDScript в выводе.
