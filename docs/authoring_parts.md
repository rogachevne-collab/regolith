# Сборка детали (Part authoring) — одна нода

Одна нода = одна деталь. На ней: модель, тип, пара понятных полей и маркеры
«крепится тут». Жмёшь **Bake** — инструмент сам делает **весь архетип** .tres:
footprint, коллайдеры, массу, mount-пады, тюнинг колеса/подвески и ось привода.
Никаких .tres руками, никаких суб-ресурсов, ориентаций (0..23) и правила
«перпендикулярно штырю» — всё считается за тебя.

Файлы:
- `scripts/authoring/part_authoring_root.gd` — `PartAuthoringRoot` (@tool)
- `scripts/authoring/mount_pad_marker.gd` — `MountPadMarker` (@tool)
- `scenes/authoring/part_authoring_template.tscn` — шаблон (уже настроен как колесо)

## Общий поток: загрузил модель → получил крепления

1. `visual_scene` → своя моделька.
2. Галка **`fit_size_to_model_now`** → тул меряет габариты меша и сам ставит
   `size_cells`. Пишет в `last_bake_diagnostics`, например
   «модель 2.50×2.50×2.50 м → 5×5×5 клеток». Если модель смещена от нуля —
   скажет насколько (гадать не будет).
3. Выбираешь `part_kind` и `mount_generation` (см. таблицу).
4. Галка **`generate_mounts_now`** → маркеры появляются как **настоящие узлы**
   в дереве: выделяй, удаляй `Delete`, таскай мышкой. Роли проставлены
   догадкой по типу детали (колесу — штырь наверх, подвеске — сокет вниз).
5. Удалил лишние, подвинул нужные → `bake_now`.

### Режимы генерации (`mount_generation`)

| Режим | Маркеров на кубе 2.5 м (5×5×5) | Когда |
|---|---|---|
| `FULL_SURFACE` — вся поверхность | **0** | блоки, рамы, корпуса. Крепится всеми сторонами, маркеры не нужны вообще |
| `PER_SIDE` — по сторонам | 6 | ограничить, какие стороны крепятся; колесо/подвеска |
| `PER_CELL` — по каждой клетке | 150 | тонкий контроль. На крупных деталях замучаешься прунить |

Дефолт: `FULL_SURFACE` для «Обычной», `PER_SIDE` для Колеса/Подвески
(переключается автоматически при смене `part_kind`).

> Коллайдер всегда один коробочный на всю деталь — у куба 2.5 м это 1 форма,
> а не 125.

## Колесо за 5 шагов

1. Копируешь `part_authoring_template.tscn`.
2. `visual_scene` → своя моделька колеса.
3. `part_id` → имя (напр. `big_wheel`).
4. Поля колеса по вкусу: `wheel_radius_m`, `wheel_drive_torque_n_m`,
   `wheel_steerable`. Остальное — дефолты.
5. Маркер `MountHere` — грань, которой колесо садится в подвеску. Если модель
   крепится сверху, он уже на `+Y`; если иначе — перетащи, он прилипнет к
   ближайшей грани. Жмёшь `bake_now`.

Итог → `resources/archetypes/authored/<part_id>.tres`. Ось привода
подбирается автоматически перпендикулярно штырю. Результат/ошибки — в
`last_bake_diagnostics` (по-человечески: «Колесо: поставь ровно один маркер…»).

## Поля (в инспекторе показываются только нужные для типа)

| Поле | Что | Дефолт |
|---|---|---|
| `part_id` | имя файла архетипа | — |
| `part_kind` | Обычная / Колесо / Подвеска | Обычная |
| `visual_scene` | твоя моделька | — |
| `size_cells` | размер коробки в клетках (1 клетка = 0.5 м) | 1×1×1 |
| `mass_kg` | 0 = авто по размеру | 0 |
| `wheel_radius_m` / `wheel_drive_torque_n_m` / `wheel_steerable` | только для Колеса | 0.4 / 65 / нет |
| `suspension_travel_m` / `suspension_stiffness_n_per_m` | только для Подвески | 0.6 / 1600 |

## Подвеска

`part_kind = Подвеска`, `size_cells = (1,2,1)`, два маркера:
- обычный (`socket_kind = STRUCTURAL`) на грани, которой садится на раму;
- один с `socket_kind = WHEEL_SOCKET` на грани, куда встаёт колесо.

Bake сам выставит подвеске `wheel_socket_face` под этот маркер — колесо и
подвеска сойдутся в игре автоматически.

## Обычная деталь / рама

`part_kind = Обычная`. Без маркеров — вся внешняя поверхность становится
крепёжной (как рама, болтится со всех сторон). Хочешь точечно — поставь
маркеры `STRUCTURAL` только на нужные грани.

## Проверка

`tests/run_one.sh test_part_authoring` — headless-гейт: колесо и подвеска
собираются в **валидный** архетип, ось привода перпендикулярна штырю, «два
маркера на колесе» отклоняются, .tres сохраняется и перечитывается. Входит
в `tests/run_tests.sh`.

> Новый `@tool`-скрипт с `class_name` регистрируется кэшем классов Godot.
> Если headless-тест пишет «Identifier … not declared» — обнови кэш:
> `bash run.sh --headless --editor --quit` (или просто открой проект в редакторе).

## Пока НЕ покрыто

Двухчастные актуаторы (пистон+головка, ротор, шарнир) — другой механизм
(base авто-спавнит head через joint). Их разметку добавим отдельным типом.
